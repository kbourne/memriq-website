Memriq is operated almost entirely by autonomous agents. The CAO sets goals, reviews exceptions, and ships the occasional patch. Everyone else is software.

// CHIEF AGENT OFFICER

Keith Bourne

Founder, CAO, advisor.

Forward Deployed AI Engineer at Tribe AI, where the work is helping enterprise teams move from isolated LLM features to measurable, goal-driven agent systems. Before that: a CoALA-based agent memory system at Magnifi by TIFIN (500K+ users, $4B linked assets), and a modular AI platform at Johnson & Johnson used by tens of thousands of data scientists. Ten years shipping production AI systems; before that, a COO who scaled an organization 10×. The architecture lens and the operating-model lens are the same lens.

Memriq exists to answer one question at scale: what does an organization look like when agents are the employees? The CAO role here is deliberately minimal — set direction, review exceptions, approve what needs approving. The agents have the reins. The system is the work.

Author of Unlocking Data with Generative AI and RAG (Packt, 2 editions) — the second edition covering graph-powered RAG agents, memory pipelines, semantic caching, and agentic system design. Host of The Memriq AI Inference Brief: weekly, two editions, built for practitioners and the executives who fund them.

// AGENT LEADERSHIP

Nine agents that run the conglomerate.

These are the orchestrators, coordinators, and platform leads — the agents who route work, guard the gates, and keep the memory layer coherent. Each portfolio company has its own agent staff working under one of them.

ATLAS
CAO INTERFACE · GREEK

Atlas is the bridge between Keith and the agent system. He parses CAO commands, routes them to the right agent, and reports back what happened.

IRIS
TASK CONDUCTOR · GREEK

Iris routes incoming work across the agent system and orchestrates multi-agent workflows. She's the air traffic controller — every task reaches the right agent, every workflow completes.

HEIMDALL
ESCALATION SENTINEL · NORSE

Heimdall decides what needs CAO review and what doesn't. He classifies decisions by risk, holds the approval gates, and keeps the system from auto-approving what shouldn't be.

GENESIS
COMPANY CREATION ORCHESTRATOR · ENGLISH

Genesis runs the pipeline that turns CAO directives into new portfolio companies — classification, market analysis, ontology, handoff to the forge.

HEPHAESTUS
FORGE & PATTERN LIBRARY · GREEK

Hephaestus is the technology development center. He analyzes code patterns, generates new agents, and maintains the pattern library. The agent that builds the next agents.

AEGIS
CODE VERIFICATION COORDINATOR · GREEK

Aegis orchestrates the defensive review system that runs on every code change in Labs. He decides which sub-agents to invoke and produces a single verdict the engineering agents can act on.

HERMES

Hermes routes messages between Scout's research agents, manages the workflow state machine, and triggers reflection cycles to ensure the pipeline converges on quality.

PROMETHEUS
LABS THROUGHPUT ORCHESTRATOR · GREEK

Prometheus maximizes task completion across Labs' connected daemon machines. He keeps the queue moving, prevents idle capacity, and balances load.

MIMIR

Mimir guards the different levels of the memory platform across the conglomerate. Every agent reads and writes through him; every memory has a scope and an owner because of him.
